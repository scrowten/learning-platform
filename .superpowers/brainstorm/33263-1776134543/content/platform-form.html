What's the primary form factor?

This shapes everything — file structure, navigation, tooling, integration with your other projects.

A

Enhanced Static Repo

Same GitHub-native approach — folders, markdown, notebooks. Add a domain registry (YAML/JSON) as the meta-layer. No build step, no server. Navigation via README cross-links and a generated index.

Pros

  • Zero infrastructure
  • Already works
  • Jupyter native

Cons

  • Navigation gets unwieldy at scale
  • No search, no cross-domain links
B

Rendered Docs Site

MkDocs Material or Docusaurus — markdown stays the source of truth, but renders into a navigable web UI with search, tags, cross-links, and a dependency graph view. Deploys to GitHub Pages.

Pros

  • Great UX for navigation
  • Full-text search
  • Tag/difficulty filtering

Cons

  • Build step required
  • Notebooks need nbconvert export
C

API-First Platform

Connect to the agentic-ai backend you're building. Content is data — served via API. Your Flask+React webapp queries it. Enables AI-powered features: personalized paths, adaptive quizzes, concept Q&A.

Pros

  • AI-native learning features
  • Integrates with your webapp
  • Dynamic personalization

Cons

  • Most complex to build
  • Content needs structured schema
D

Hybrid — Docs Site Now, API Later

Start with MkDocs or Docusaurus as the human-facing layer. Design the content schema (frontmatter, metadata) to be API-ready. Phase 2: expose a content API consumed by agentic-ai and your webapp.

Pros

  • Ship something usable fast
  • Content schema is future-proof
  • Clear phased roadmap

Cons

  • Schema design matters upfront
