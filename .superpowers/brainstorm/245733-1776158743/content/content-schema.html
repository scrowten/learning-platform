Content Schema Design

Two files define the platform's intelligence layer: domain-registry.yaml and the per-module frontmatter in README.md. The sync worker reads these to build the DB.

1 — domain-registry.yaml (repo root)

domains:
  - id: ds-ml
    name: "Data Science & ML Engineering"
    path: domains/ds-ml
    color: "#3b82f6"
    icon: "🧠"
    tags: [quantitative, programming, math]

2 — Module README.md frontmatter (every module)

---
id: supervised-learning

title: "Supervised Learning"

  - linear-algebra

  - gradient boosting dominance on tabular data
  - AutoML (H2O, AutoSklearn)

# Supervised Learning
...rest of README...

Same frontmatter works for any domain. An economics module would be domain: economics, a trading module domain: trading. The sync worker is domain-blind — it reads frontmatter, not folder names.

3 — Module directory structure (universal template)

DS/ML module (existing)
supervised/
├── README.md        ← frontmatter here
├── concepts.md      ← theory, math, proofs
├── cheatsheet.md    ← quick reference
├── sota.md          ← current field state
└── notebooks/
    ├── 01_from_scratch.ipynb
    └── 02_real_dataset.ipynb
Economics module (new)
supply-demand/
├── README.md        ← same frontmatter
├── concepts.md      ← theory, models
├── cheatsheet.md    ← quick reference
├── sota.md          ← current debates
└── notebooks/
    ├── 01_model.ipynb      ← optional
    └── 02_real_data.ipynb  ← optional

Notebooks are optional per domain. Economics/finance modules may use case studies or spreadsheets instead. The template stays consistent; the file types adapt.
