Database Schema

PostgreSQL. 7 tables — each maps directly to a feature. Sync worker populates the top 4; runtime populates the bottom 3.

domains — from domain-registry.yaml
id varchar PK
name varchar
path varchar
color varchar
icon varchar
tags text[]
created_at timestamptz
modules — from README.md frontmatter
id varchar PK (slug)
domain_id → domains.id
title varchar
difficulty varchar
tags text[]
estimated_hours int
git_path varchar
last_reviewed date
sota_topics text[]
synced_at timestamptz
module_prerequisites — prerequisite graph
module_id → modules.id
prereq_id → modules.id
PK: (module_id, prereq_id)

Enables topological sort for learning paths and skip-graph queries.

content_chunks — for Concept Q&A (RAG)
id uuid PK
module_id → modules.id
source_file varchar (concepts.md etc)
chunk_index int
content text
embedding vector(1536) (pgvector)
synced_at timestamptz

pgvector extension. Sync worker chunks concepts.md → embeds via agentic-ai model router.

quiz_items — Auto-Generated Quizzes
id uuid PK
module_id → modules.id
question text
options jsonb (MCQ choices)
correct_answer varchar
explanation text
difficulty varchar
generated_at timestamptz
user_progress — Progress Tracking
user_id varchar
module_id → modules.id
status varchar (not_started/in_progress/done/skipped)
started_at timestamptz
completed_at timestamptz
quiz_score float
PK: (user_id, module_id)
sota_reviews — SOTA Alert Tracking
id uuid PK
module_id → modules.id
status varchar (current/stale/flagged)
flagged_topics text[]
checked_at timestamptz
draft_update text (AI-drafted suggestion)

Periodic worker compares sota.md topics against arxiv/news. Sets status=flagged and writes a draft update when content appears stale. Cleared when module is re-reviewed.

Synced from content repo
